A fundamental aspect of the TO is the distinction between the physical densities and the merely numerical ones x.

Here, , called filter radius, is the radius of the specified neighborhood and the distance between the centroids of the two elements. This filter makes the results independent from the adopted mesh, improving the resolution as decreases. However, the maximal resolution is linked to the mesh itself, since the minimum applicable cannot be smaller than elements' size.

We have to take into account this filtering both on the evaluation of the derivatives and on the FEA .

In fact, the MMA is the optimizer we adopt: this uses the values and the derivatives with respect to the numerical densities.

When we pass to the finite elements analysis, we have to consider the physical and not the numerical x.

When we have to compute the derivatives with respect to the numerical densities, we can proceed as follows: first we consider the derivatives with respect to the , then we apply the chain rule, by multiplying the result by the matrix .

%% Mesh-independency filter preparation

rmin = 6; %depends on the mesh-size (it's not a contradiction with the section title)

xG = (x1+x2+x3)/3;

yG = (y1+y2+y3)/3;

i=1:nt;

[I,J]=meshgrid(i,i);

W = max(0,rmin-sqrt((xG(J)-xG(I)).^2+(yG(J)-yG(I)).^2));

W = W./repmat(sum(W,2),1,nt);

W = sparse(W);

Objective function

The objective function in our case is th volume fraction which has to be minimized. It doesn't depends on the free nodal displacements, so its related Lagrangian multiplier will be zero.

However, we use this function to compute the residual and its derivative with respect to the density.

%function [ o, do, U_15, dR_drho, K_15 ] = objective_fcn_fext( x, W, penal, ddl, AT, AA, B, a1, a2, a3, b1, b2, b3, mu0, lambda0, h, imposed_dofs, ind, free_dofs )

%The objective function is the compliance due to an external force

% Since the problem is nonlinear, each single part of the calculus of the

% derivatives depends on the nodal displacements. The derivative of the

% compliance respect to the displacements is the internal force. Then, we

% use the adjoint sensitivity method adopted in Bhattacharyya's article.

rho = W*x; % filtering

o = sum(rho.*AT)/sum(AT);

do = AT/sum(AT); % derivative related to the rho

mue = mu0 *rho.^penal; %expression provided by Bhattacharyya

lambdae = lambda0 *rho.^penal;

[ U_15, K_15, ~, ~] = nonlinear_tri_fext( ddl, AT, AA, B, a1, a2, a3, b1, b2, b3, mue, lambdae, h, imposed_dofs, ind );

%o = Fint_15(2*ind-1); % compliance

%

q = U_15(ddl); % assignment of each degree of freedom to each element

% Scomposition of the matrix q: in each row there is the list of

% displacements for each element. The number of rows gives the

% number of triangles in the mesh.

q1 = q(:,1);

q2 = q(:,2);

q3 = q(:,3);

q4 = q(:,4);

q5 = q(:,5);

q6 = q(:,6);

% J is the determinant of the deformation tensor and it's computed

% for each element.

J = (b1.*q1+b2.*q3+b3.*q5).*(a1.*q2+a2.*q4+a3.*q6)/4./(AT.^2)-(a1.*q1+a2.*q3+a3.*q5).*(b1.*q2+b2.*q4+b3.*q6)/4./(AT.^2) + (a1.*q2 + a2.*q4 + a3.*q6 + b1.*q1 + b2.*q3 + b3.*q5)/2./AT + 1;

q=q.';

m = length(U_15);

nt = length(a1); % n is the number of elements (triangles)

dFint_drho = sparse(m,nt);

% The force assembly is the same used in gradientfcn2_fext

for i=1:nt

I = ddl(i,:);

Fel=1/J(i)*(lambda0*log(J(i))-mu0)*([b1(i) a1(i) b2(i) a2(i) b3(i) a3(i)]/2/AT(i)+(q(:,i).')*AA(:,:,i))+mu0*([b1(i) a1(i) b2(i) a2(i) b3(i) a3(i)]/2/AT(i)+(q(:,i).')*B(:,:,i));

dFint_drho(I,i)= (Fel.')*AT(i)*h*rho(i)^(penal-1)*penal;

end

% We use the adjoint sensitivity method adopted in Bhattacharyya's article

dR_drho = dFint_drho;

%dR_drho(2*ind-1,:) = dR_drho(2*ind-1,:)-do_drho;

dR_drho = dR_drho(free_dofs,:);

do = W*do; % density filtering

%end

Constraints function

Our constraints are given by the stress and the displacement at the trailing edge.

In order to consider the stress constraint as the preferential one, we can re-scale it, just multiplying it by 2.

%function [ c, ceq, dc, dceq, sigma ] = constraints_fcn2( x, W, AT, tail_part,t,p,ind_borde_fuite, U_15, dR_drho, K_15, free_dofs, U0, sigmalim, nt,lambda0, mu0, a1, a2, a3, b1, b2, b3, ddl )

%This function returns the value of the constraints used in the MMA

%algorithm or in the fmincon (for the minimization problem).

%The first constraint is the volume fraction, which depends just on the

%densities rho.

rho = W*x; % filtering

% The second constraint is the displacement of the trailing edge, which

% should be greater (in module) than a certain U0: since U0<0, we will have

% U <= U0 ------> 1 - U/U0 <= 0

[ GKS, dGKS_dUf, dGKS_drho, sigma ] = GKS_fcn2( U_15, ddl, free_dofs, lambda0, mu0, a1, a2, a3, b1, b2, b3, AT, rho, sigmalim, nt );

c(1) = 2*GKS;

lambda = -dGKS_dUf*(K_15(free_dofs,free_dofs)\eye(length(free_dofs)));

dc(:,1) = 2*(dGKS_drho + (lambda*dR_drho).');

c(2) = (1-U_15(2*ind_borde_fuite)/U0);

% We use the adjoint sensitivity method adopted in Bhattacharyya's article

dUe_dUf = sparse(1,2*ind_borde_fuite,-1/U0,1,length(U_15));

dUe_dUf = dUe_dUf(free_dofs);

lambda = -dUe_dUf*(K_15(free_dofs,free_dofs)\eye(length(free_dofs)));

dc(:,2) = (lambda*dR_drho).';

dc = W*dc; % filtering of derivatives to bring them back to be dc/dx

ceq = [];

dceq = [];

%end
